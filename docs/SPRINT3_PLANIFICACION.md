# Sprint 3 - Planificacion

## Objetivo del sprint
Implementar un dashboard gerencial para soporte decisional e integracion final de modulos, cumpliendo HU-06 y HU-18.

## Historias de usuario del Sprint 3
- HU-06 (Could Have, 3 SP): Ver reporte de ventas diarias con filtros por fecha y total vendido.
- HU-18 (Could Have, 3 SP): Consultar historial de ventas por fecha y mostrar resultados.
- HU-07 (Won't Have, 13 SP): Aplicacion movil (no se desarrolla en este sprint).
- HU-21 (Could Have, 3 SP): Recuperar contrasena (diferida).

## Alcance tecnico de Actividad 7
1. Dashboard con 5+ KPIs.
2. Visualizaciones con 3+ tipos de graficos (linea, pie, barras).
3. Endpoint `GET /dashboard/kpis` con KPIs agregados.
4. Filtros dinamicos por rango de fechas con `fetch`.
5. Integracion de modulos en flujo completo por roles.
6. 10+ casos de prueba de integracion documentados.

## Entregables del sprint
- Endpoint JSON para KPIs.
- Vista dashboard actualizada con Chart.js.
- Historial de ventas con busqueda por fecha (HU-18).
- Documento de pruebas de integracion.
- Guion de Sprint Review final y retrospectiva.

- ODS 9: Industria, innovacion e infraestructura.
