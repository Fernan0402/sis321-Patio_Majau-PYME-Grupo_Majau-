# Guía de Entrevista — Operación Restaurante

## Objetivo

Levantar necesidades reales de operación para validar HU y mejoras del sistema.

## Perfil de entrevistados

- Propietario/Administrador
- Mesero
- Cajero
- Encargado de cocina

## Preguntas base

1. ¿Qué problemas frecuentes tienen al registrar pedidos?
2. ¿Dónde se generan más errores: cocina, caja o inventario?
3. ¿Cómo controlan actualmente faltantes de insumos?
4. ¿Qué información necesita el administrador al cierre del día?
5. ¿Qué campos son obligatorios en factura para sus clientes?
6. ¿Qué tipo de reportes usan para decidir compras?
7. ¿Qué paso del proceso actual toma más tiempo?

## Evidencias a recopilar

- Flujo actual (manual/digital)
- Formatos usados en caja
- Políticas de stock mínimo
- Casos de error frecuentes

## Resultado esperado

- Ajustar backlog para Sprint 2+
- Refinar criterios de aceptación por HU
- Reducir retrabajo técnico en siguientes iteraciones
